Accessibility

Readable, simple, and still improving.

VaporHuman should be easy to read, navigate, review, and correct.

Accessibility posture

The site should be readable, navigable, and honest about its limits.

VaporHuman is built as static HTML and CSS with semantic headings, text-first content, visible links, keyboard-friendly navigation, and no tracking or third-party embeds.

This is not a formal conformance certification. It is a current posture and improvement target for a small static site.

Practices

Current accessibility-oriented choices

Semantic structure

Pages use headings, landmarks, lists, and descriptive link text where practical.

No hidden interactions

There are no forms, popups, account flows, or client-side app controls.

Readable defaults

The design favors large type, strong contrast, and responsive layouts.

Reviewable content

The site is intentionally simple so issues can be found and corrected.

Known limits

Feedback path is not public yet.

No public email or contact form is active yet. Until the email lane is verified and approved, accessibility feedback collection is not published on this site.

VaporHuman is an early static trust surface. Site copy, page structure, availability, and future features may change as the project develops.

This site does not provide medical, legal, financial, employment, identity, emergency, or professional advice. Assistant output should support human judgment, not replace it.

No forms, login, checkout, payment flow, analytics, tracking scripts, contact capture, public email address, donation flow, or live assistant service are active here.
