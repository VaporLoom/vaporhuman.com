Accessibility

Readable, simple, and still improving.

VaporHuman should be easy to read, navigate, review, and correct.

Accessibility posture

The site should be readable, navigable, and honest about its limits.

VaporHuman is built with static output, semantic headings, text-first content, visible links, keyboard-friendly navigation, and no tracking or third-party embeds. The work-intake path opens a hosted Tally form.

This is not a formal conformance certification. It is a current posture and improvement target for a small site.

Practices

Current accessibility-oriented choices

Semantic structure

Pages use headings, landmarks, lists, and descriptive link text where practical.

No hidden interactions

There are no embedded forms, popups, account flows, or client-side app controls.

Readable defaults

The design favors large type, strong contrast, and responsive layouts.

Reviewable content

The site is intentionally simple so issues can be found and corrected.

Known limits

Feedback path is not public yet.

No public email address is active yet. The Tally work-intake form is not a dedicated accessibility feedback channel.

VaporHuman designs private AI continuity systems. Site copy, page structure, availability, and future features may change as the product lane develops.

This site does not provide medical, legal, financial, employment, identity, emergency, or regulated advice. Assistant output should support human judgment, not replace it.

The work-intake path uses a hosted Tally form, and paid discovery uses a hosted Stripe Payment Link. This site does not embed forms, login, checkout, public email address, public support flow, subscriptions, or live assistant service. This build does not include analytics or tracking scripts.
